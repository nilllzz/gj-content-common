Game Jolt defines a list of contexts for each Content Document to use, from which the capabilties can be infered.

## Capabilities explained

Each of the mentioned capabilities above allow a certain set of [\<type:content-object\>](/docs/types/content-object.md) and [\<type:mark-object\>](/docs/types/mark-object.md) in a Content Document.

By default, each context allows the usage of the [\<object:paragraph\>](/docs/objects/paragraph.md) and [\<object:text\>](/docs/objects/text.md) objects.

### Text decoration capabilities

| Capability           | Allowed things                                                                                                                             |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| Bold Text            | [\<mark:strong\>](/docs/marks/strong.md) on Text objects                                                                                   |
| Italic Text          | [\<mark:italic\>](/docs/marks/italic.md) on Text objects                                                                                   |
| Link Text            | [\<mark:link\>](/docs/marks/link.md) on Text objects<br>However, without the "Custom links in Text" capability, only autolinks are allowed |
| Code in Text         | [\<mark:code\>](/docs/marks/code.md) on Text objects                                                                                       |
| Strikethrough Text   | [\<mark:strike\>](/docs/marks/strike.md) on Text objects                                                                                   |
| Tag                  | [\<mark:tag\>](/docs/marks/tag.md) on Text objects, allowing for the usage of hashtags.                                                    |
| Mention              | [\<mark:mention\>](/docs/marks/mention.md) on Text objects, allowing for the usage of user account mentions.                               |
| Custom links in Text | Allows links with `href` independent of the text they are decorating. See [\<mark:link\>](/docs/marks/link.md) for more.                   |
